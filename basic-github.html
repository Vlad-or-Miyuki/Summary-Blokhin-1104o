Основы GitHub

Что такое GitHub?

- Объяснение: GitHub - это платформахостинга, хранения и совместной разработке. Здесь разработчики могутхранить свои проекты, работать над ними в команде,самостоятельно,параллельнои отслеживать изменения и оновления. - Аналогия: GitHub, как облачное хранилищепроектов, подобен социальной сети для разработчиков, где они вместе строят исовершенствуют свои творения.

SSH-Ключ

SSH-ключ: - Объяснение: SSH-ключ - это способ безопасной аутентификации(подключения)на удаленном(облачном)сервере. Он используется для подключения к GitHub,например, без постоянного ввода пароля. - Аналогия: SSH-ключ, как сканер отпечатка пальца, позволяет вам разблокировать устройство (подключаться к серверу) безнеобходимости запоминать пароль.

Как сгенерировать SSH-ключ и добавить к себе на GitHub:

Откройте терминал и выполните:

ssh-keygen -t rsa -b 4096

Нажмите Enter, чтобы принять стандартное расположение файла. Введите пароль или нажмите Enter, чтобы оставить его пустым

В терминале выполните:

cat ./.ssh/id_rsa.pub

Перейдите в настройки GitHub -> SSH and GPG keys -> New SSH key -> вставьте ключ.

Что такое Репозиторий?

Репозиторий: - Объяснение: Репозиторий - это место, где хранится код проекта, вместе со всейего историей изменений. - Аналогия: Репозиторий, как папка спроектом, подобен архиву, где хранятся все чертежи, планы и изменения к вашемуглавному творению.

Как добавить к себе чей-то репозиторий: Шаги: Скопируйте URL репозитория с GitHub (зеленая кнопка Code).

В терминале выполните:

git clone URL Репозитория

Что такое Коммит?

- Объяснение: Коммит - это сохранение изменений в коде проекта. Он создаетточку в истории, которую можно сохранить и вернуться к ней. - Аналогия: Коммит, как сохранение в игре, фиксирует текущее состояние, чтобыв случае чего можно было вернуться к нему.

Как сохранить изменения на GitHub (сделать коммит):

git add -A | git add - -all

Чтобы добавить все изменения.

git commit -m 'Описание коммита'

Далее выполните:

git push

чтобы отправить изменения на GitHub.

Что такое Ветка?(Версия)

Ветка: - Объяснение: Ветка - это отдельнаялиния разработки, которая может быть создана для работы над новойверсией(функцией) или исправлением, не затрагивая основной код проекта. - Аналогия: Ветка, как отдельный поток в разговоре, позволяет вам обсуждать ивносить изменения, не прерывая основной диалог.

Как создать ветку:

Выполните:

git branch task1

Или выполините:

git checkout -b task1

Эта комманда не только создаст ветку но и сразу на нее переключится.

Как открыть VSCode из терминала Linux?

В терминале выполните:

Как запустить программу в терминале (через Node.js):

Введите в консоли VS Code или Терминале Linux:

node имя файла.js

Часто задаваемые вопросы:

1. В каком случае нужно генерировать новый SSH-ключ:

  1. Когда вы впервые начинаете работу с GitHub.
  2. При использовании нового компьютера или сервера для работы с репозиториями.
  3. Когда старый ключ утерян или скомпрометирован.

1.1 Могу ли я использовать один и тот же SSH ключ на разных устройствах:

      Да, вы можете использовать один и тот же SSH ключ на разных устройствах. Это удобно при работе с несколькими устройствами.

2. В каких случаях надо использовать git config user.name (user.email) <аргументы>:

  1. При первой настройке Git на новом устройстве.
  2. При изменении вашего имени или электронной почты.
  3. Когда вам нужно использовать разные имена или адреса электронной почты для разных проектов.

3. В каком случае нужно создавать ветку и для чего она нужна:

  1. Новая функциональность: создайте ветку для работы над новой функцией, чтобы не затрагивать основной код.
  2. Исправление ошибок: создайте ветку для исправления ошибок, чтобы избежать конфликтов с основной разработкой.
  3. Разработка параллельных версий: создайте ветку для разработки и тестирования новых идей без влияния на стабильный код.

4. Что писать в комментарии для git commit:

  1. Краткое, но информативное описание изменений, чтобы понять суть коммита.
  2. Укажите, какие задачи решает этот коммит.
  3. Следуйте принципу "Как" и "Почему" для более полного объяснения.

5. Что такое Pull Request и зачем он нужен:

  1. Pull Request (или Merge Request) - это запрос на внесение изменений из вашей ветки в основную ветку проекта.
  2. Используется для обсуждения и рецензии ваших изменений перед их объединением с основным кодом.
  3. Позволяет совершать изменения согласованно и избегать конфликтов при слиянии кода.
